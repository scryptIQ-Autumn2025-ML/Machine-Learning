Machine Learning 2 - Glossary

Glossary ML2
Accuracy: The proportion of correct predictions made by a classifier across all samples. While intuitive, accuracy can be misleading for imbalanced datasets and should not be used in isolation.
Class Imbalance: A situation where one class occurs much more frequently than another. Class imbalance can bias classifiers and distort evaluation metrics such as accuracy.
Confusion Matrix: A tabular summary of classification outcomes showing true positives, true negatives, false positives, and false negatives. It provides a detailed view of where a model is making errors.
Decision Boundary: The surface in feature space that separates different predicted classes. Decision boundaries help visualise how a classifier partitions the data.
F1 Score: A single metric that combines precision and recall using their harmonic mean. It is particularly useful when false positives and false negatives are both important.
False Negative: A case where the model incorrectly predicts the negative class when the true label is positive. False negatives are especially critical in medical and biological applications.
False Positive: A case where the model incorrectly predicts the positive class when the true label is negative. High false positive rates can lead to misleading conclusions.
Hyperparameter: A configuration value set before training that controls how a model learns, such as regularisation strength or kernel parameters. Hyperparameters differ from learned model parameters.
Model Refinement: The iterative process of improving model performance through techniques such as normalisation, hyperparameter tuning, and repeated evaluation.
Normalisation: A preprocessing step that rescales features so they contribute comparably during training. Normalisation can significantly affect distance-based and optimisation-based models.
Permutation Test: A statistical method that evaluates whether a model’s performance is better than chance by repeatedly shuffling labels and retraining the model.
Permutation Test Score: The distribution of scores obtained from permuted labels, used to assess the statistical significance of a classifier’s observed performance.
Precision: The proportion of positive predictions that are correct. Precision reflects how reliable positive predictions are.
Recall (Sensitivity): The proportion of true positive samples that are correctly identified. Recall measures a model’s ability to detect positive cases.
ROC Curve (Receiver Operating Characteristic Curve): A curve showing the trade-off between true positive rate and false positive rate across different decision thresholds.
ROC AUC (Area Under the Curve): A scalar summary of the ROC curve representing the probability that a randomly chosen positive sample is ranked higher than a negative one.
State-Space Visualisation: A visualisation technique where predictions are made across a grid of feature values to reveal classifier behaviour across the entire feature space.
Statistical Significance (Model Performance): An assessment of whether observed model performance is unlikely to have arisen by chance, often evaluated using permutation testing.
Threshold: A decision value applied to model scores or probabilities that determines class assignment. Changing the threshold alters precision-recall trade-offs.
Train–Test Split: A data partitioning strategy that separates training data from evaluation data to provide an unbiased estimate of model performance.
Toy Dataset: A synthetic dataset designed for controlled experimentation and visualisation. Toy datasets are useful for learning but should not be over-interpreted.
True Negative: A case where the model correctly predicts the negative class.
True Positive: A case where the model correctly predicts the positive class.
