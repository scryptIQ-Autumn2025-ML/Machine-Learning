    Introduction ML2

    In ML2, we aim to consolidate many of the concepts covered in the previous machine learning lesson, and build upon your understanding of supervised classification by focusing on how models are evaluated, compared and refined, in practice. Building on earlier work with classifiers and decision boundaries, we will take a structured approach to assessing model performance using quantitative metrics and known validation techniques. Rather than treating model accuracy as a single outcome, we will examine how different evaluation tools reveal different aspects of a classifier’s behaviour, particularly when working with noisy, non-linearly separable data.

    We begin by exploring synthetic datasets to explore classification boundaries, examine the strengths and limitations of different metrics, and learn how to interpret model outputs beyond raw predictions. You will use confusion matrices, ROC and precision-recall concepts, and permutation testing to assess whether observed performance is meaningful, or due to chance. We then look at practical refinement techniques, including normalisation and hyperparameter tuning, highlighting the balance between improving performance and avoiding overfitting, where a model can fit the training set too well, and lose its ability to generalise meaningfully on test data. By the end of this lesson, you should be able to evaluate classification models critically, justify modelling choices, and apply principled refinement strategies in applied machine learning contexts.

    State-Space Visualisation Extends scatter plots to decision boundaries Meshgrid prediction over feature space Visualise how classifiers separate classes Assuming smoother boundaries imply better generalisation
    Train–Test Split Introduces unbiased evaluation practice train_test_split() Separate training and evaluation data Data leakage between training and test sets
    Model Accuracy Establishes baseline performance assessment .score(), accuracy_score() Quantify overall prediction correctness Treating accuracy as sufficient on its own
    Recall and Precision Deepens evaluation beyond accuracy recall_score(), precision concepts Assess sensitivity to positive classes Ignoring class imbalance effects
    F1 Score Combines precision and recall f1_score() Balance false positives and false negatives Misusing F1 when class priorities differ
    Confusion Matrix Visual diagnostic of prediction outcomes confusion_matrix() Identify systematic classification errors Reading raw counts without normalisation
    Permutation Testing Adds statistical validation of model performance permutation_test_score() Test whether performance exceeds chance Using too few permutations for robust inference
    Cross-Validation and Repetition Reduces variance in evaluation Cross-validators within permutation testing Improve robustness of performance estimates High computational cost if poorly configured
    Normalisation Prepares data for fair model comparison Feature scaling and norm-based normalisation Improve training stability and comparability Applying normalisation without understanding data structure
    Hyperparameter Tuning Refines model performance Classifier hyperparameters Improve results while maintaining generalisability Overfitting through excessive tuning
    Progressive Refinement Integrates evaluation and optimisation Iterative training and testing Systematically improve model performance Optimising metrics without validating assumptions
